Security Policy

Last updated: June 2025

Security is fundamental to REHABION. We handle sensitive healthcare technology and take the protection of data, systems, and research infrastructure seriously. If you discover a security vulnerability, please report it responsibly.

1. Our Security Practices

We implement the following security measures across our website and systems:

  • HTTPS everywhere â€” all traffic is encrypted using TLS
  • Content Security Policy (CSP) â€” restricts which scripts and resources can be loaded
  • Anti-spam protection â€” honeypot fields on all contact forms to block bots
  • Input sanitisation â€” all user input is sanitised before processing or display
  • No patient data on web infrastructure â€” clinical data is never stored on public-facing servers
  • Dependency minimisation â€” we use no client-side JavaScript frameworks that expand our attack surface
  • Git secret protection â€” API keys and credentials are never committed to version control

2. Responsible Disclosure

If you discover a security vulnerability in any REHABION system, website, or product, we ask that you:

  • Report it privately to us before making it public
  • Give us reasonable time (30 days) to investigate and remediate before disclosure
  • Not exploit the vulnerability or access data beyond what is necessary to demonstrate the issue
  • Not conduct denial-of-service attacks or automated scanning without prior approval

Report vulnerabilities to:

muhammadrmzn84@gmail.com

3. What We Will Do

Upon receiving a responsible disclosure report, REHABION will:

  • Acknowledge receipt within 48 hours
  • Investigate the report and confirm whether a vulnerability exists
  • Work to remediate confirmed vulnerabilities as a priority
  • Keep you informed of our progress
  • Credit you in our security acknowledgements (if you wish)

4. Scope

This policy applies to:

  • The REHABION website (rehabion.com.pk)
  • Any REHABION web applications or APIs currently in development
  • The GitHub repositories under the REHABION organisation

Out of scope: third-party services we use (GitHub, Formspree, Google). Report those directly to the respective providers.

5. Data Protection Standards

REHABION aligns with the following standards for handling sensitive data:

  • ISO 27001 information security management principles
  • GDPR principles for any data involving EU residents
  • Pakistan's Electronic Data Protection Bill (and applicable PEMRA guidelines)
  • National Bioethics Committee (NBC) Pakistan guidelines for research data

6. No Bug Bounty Programme

REHABION does not currently operate a paid bug bounty programme. We are a research-stage startup and cannot offer financial rewards at this time. However, we genuinely appreciate responsible security research and will publicly credit researchers who help us improve our security posture.
